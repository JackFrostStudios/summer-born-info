# Roadmap Instructions

Use this folder for roadmap artifacts that describe high-level direction, milestones, assumptions, and open questions.

## Skill

Use the [`roadmap-author` skill](../.codex/skills/roadmap-author/SKILL.md) for the roadmap workflow, including requirement probing, milestone design, and roadmap structure.

## Folder Rules

- Keep roadmap documents high level and outcome focused.
- Prefer one document per initiative unless there is a clear reason to split it.
- Record assumptions and open questions explicitly when inputs are incomplete.
- Keep detailed implementation plans, sprint tasks, and technical design notes outside this folder unless the roadmap needs to link to them.

## Expected Output

Roadmaps in this folder should normally include goals, non-goals, high-level requirements, milestones, dependencies, risks, assumptions, and open questions.
