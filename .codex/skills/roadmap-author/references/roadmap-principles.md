# Roadmap Principles

Use these notes when building a roadmap skill, a roadmap document, or a requirements-discovery prompt.

## Prompt Design

- OpenAI prompting guidance recommends being explicit about the task and output format, then refining instructions with examples when needed.
  Source: https://developers.openai.com/api/docs/guides/prompting
- OpenAI structured outputs guidance recommends defining a schema when the response must consistently include required fields.
  Source: https://developers.openai.com/api/docs/guides/structured-outputs
- Anthropic prompt guidance recommends writing instructions clearly enough that a colleague with minimal context could follow them, and using sequential steps when order and completeness matter.
  Source: https://platform.claude.com/docs/en/build-with-claude/prompt-engineering/claude-prompting-best-practices#be-clear-and-direct

## Roadmap and Milestones

- Linear describes milestones as stages in a project's lifecycle and emphasizes tracking milestone progress in project and timeline views.
  Source: https://linear.app/docs/project-milestones
- Atlassian's roadmap guide highlights that development roadmaps should show prioritized customer value, target release dates, and milestones, while detailed delivery planning should remain separate.
  Source: https://www.atlassian.com/agile/product-management/product-roadmaps

## Turning Plans Into Trackable Work

- GitHub issue forms are useful when contributors need to provide specific, structured information.
  Source: https://docs.github.com/en/communities/using-templates-to-encourage-useful-issues-and-pull-requests/about-issue-and-pull-request-templates
- GitHub planning guidance recommends breaking large initiatives into issues and sub-issues so work can be tracked at the right level of detail.
  Source: https://docs.github.com/en/issues/tracking-your-work-with-issues/learning-about-issues/planning-and-tracking-work-for-your-team-or-project
